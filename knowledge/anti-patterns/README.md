# Anti-patterns

> **Pièges récurrents à proscrire** — « jamais ainsi, parce que… ». Le miroir des
> [`patterns/`](../patterns/) : on documente l'erreur *et* sa raison *et* l'alternative.

**Y va** : un piège observé plus d'une fois (ex. « `service_role` exposé au client »,
« `useEffect` pour fetch côté serveur », « valeurs magiques hors tokens »).
**N'y va pas** : une simple préférence, une règle agent (→ Skill), une exigence (→ Standard).

- **Nommage** : `kebab-case.md`.
- **Structure attendue** : *le piège → pourquoi c'est dangereux → l'alternative (lien pattern)*.
- **Lien** : référencer le Skill/Standard qui l'interdit déjà, sans le recopier.
