# Checklists

> **Portes de vérification** avant/après une action — « confirmer que… ✔ ». La distillation
> opérationnelle des Standards et des Playbooks.

**Y va** : une liste de contrôle actionnable (ex. pré-PR, mise en prod, sécurité, a11y, SEO, perf).
**N'y va pas** : la procédure complète (→ [`../playbooks/`](../playbooks/)), l'exigence normative
elle-même (→ [`../../standards/INDEX.md`](../../standards/INDEX.md)).

- **Nommage** : `kebab-case.md` (ex. `pre-pr.md`, `mise-en-prod.md`).
- **Source** : [`../../docs/scorecard.md`](../../docs/scorecard.md) est le prototype existant (10 dimensions) — à rattacher/relier ici.
- **Single-source** : une checklist **référence** le Standard qu'elle vérifie, ne le recopie pas.
