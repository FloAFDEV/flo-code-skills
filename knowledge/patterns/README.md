# Patterns

> **Solutions réutilisables éprouvées** — « voici comment faire ». Descriptif, non impératif
> (le *prescriptif* vit dans les Skills ; la *norme* dans les Standards).

**Y va** : une solution récurrente validée (ex. « auth Supabase SSR », « accès données typé »,
« structure App Router »).
**N'y va pas** : une règle impérative agent (→ Skill), une exigence normative (→ Standard),
un piège (→ [`../anti-patterns/`](../anti-patterns/)), du code exécutable (→ Tier 4 Artifacts).

- **Nommage** : `kebab-case.md` (ex. `auth-supabase-ssr.md`).
- **Appariement** : tout pattern peut avoir son anti-pattern miroir.
- **Promotion** : passe par le [flywheel](../flywheel.md) ; `provisional` tant que la porte de maturité n'est pas franchie.
