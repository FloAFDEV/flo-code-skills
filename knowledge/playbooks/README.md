# Playbooks

> **Procédures pas-à-pas pour scénarios récurrents** — « pour faire X : 1, 2, 3 ». Méthodologie
> reproductible (mise en place, migration, incident, release…).

**Y va** : une séquence d'étapes rejouable (ex. « ajouter un module CRM », « brancher Supabase sur un
nouveau projet », « répondre à un incident prod »).
**N'y va pas** : une porte de vérification seule (→ [`../checklists/`](../checklists/)),
une explication conceptuelle (→ guide/documentation), une règle agent (→ Skill).

- **Nommage** : `kebab-case.md`.
- **Relation** : un playbook se *distille* souvent en une checklist (les étapes critiques deviennent des cases à cocher).
